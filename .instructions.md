# Sahla AI - Project Instructions

## Overview
Sahla AI is a modern AI-powered marketplace platform built with Next.js 15, Supabase, and OpenAI integration.

## Tech Stack
- **Frontend**: Next.js 15 (App Router) + React 19 + TypeScript + TailwindCSS
- **State Management**: Zustand
- **Backend**: Supabase (Auth, Database, Storage, Realtime)
- **AI**: OpenAI API via Supabase Edge Functions
- **Animations**: Framer Motion
- **UI Components**: Shadcn UI + Custom components
- **Notifications**: Sonner

## Key Implementation Details

### Authentication
- Supabase Auth with Google OAuth provider
- Email/Password authentication
- Protected routes via middleware
- Session stored in Zustand store

### AI Chat System
- Conversational interface (no traditional forms)
- Progressive data collection during chat
- OpenAI integration for natural conversations
- Chat history stored in Supabase

### Admin Panel
- Restricted to: ca.markode@gmail.com (enforced via RLS + frontend checks)
- User management, job moderation, analytics
- Admin activity logging

### Database Structure
- **users**: Core user accounts with roles
- **profiles**: Extended user profile information
- **jobs**: Job listings with status management
- **ai_conversations**: Chat history and collected data
- **admin_logs**: Admin activity tracking
- Row-level security policies for data isolation

### Dark/Light Mode
- Theme stored in localStorage
- System preference detection
- Smooth transitions
- Zustand store for state management

## Critical Files

### Configuration
- `.env.local` - Environment variables (create from template)
- `next.config.ts` - Next.js configuration
- `tsconfig.json` - TypeScript configuration
- `tailwind.config.ts` - Tailwind CSS configuration

### Core Files
- `lib/supabase.ts` - Supabase client initialization
- `lib/store.ts` - Zustand stores (auth, theme, UI)
- `types/index.ts` - TypeScript type definitions
- `components/providers/theme-provider.tsx` - Theme context

### Pages Structure
- `/` - Landing page with animations
- `/auth/login` - Login page
- `/auth/register` - Registration page
- `/dashboard/chat` - AI Chat interface
- `/jobs` - Jobs listing
- `/jobs/[id]` - Job details
- `/jobs/post` - Post new job
- `/admin` - Admin dashboard (restricted)

## Important Notes

### Styling Approach
- TailwindCSS for all styling (utility-first)
- Dark mode via `dark:` prefix
- Custom components in `components/ui/`
- Responsive design with `md:` and `lg:` breakpoints

### State Management
- Zustand stores in `lib/store.ts`
- `useAuthStore` - User authentication state
- `useThemeStore` - Theme preference
- `useUIStore` - UI state (sidebar, etc.)

### Component Organization
- Reusable components in `components/`
- Shadcn UI components in `components/ui/`
- Page-specific components co-located with pages
- Prefer composition over inheritance

### API Integration
- Supabase client library for direct database access
- OpenAI API calls via Edge Functions (not direct)
- Error handling with try-catch and toast notifications
- Loading states with spinners

### Security Considerations
- Environment variables never exposed to client (except NEXT_PUBLIC_*)
- RLS policies enforce data isolation
- Admin checks both frontend and backend
- Input validation on form submissions

## Development Workflow

### Adding a New Page
1. Create folder under `app/`
2. Add `page.tsx` file
3. Use `'use client'` if interactive
4. Import types from `@/types`
5. Style with TailwindCSS classes

### Adding a New Component
1. Create in `components/` folder
2. Export as named export
3. Accept props with TypeScript interfaces
4. Use Zustand stores for state if needed
5. Add error handling and loading states

### Adding a Database Table
1. Create table in Supabase console
2. Add type definition in `types/index.ts`
3. Create API functions in `lib/api/`
4. Add RLS policies for security
5. Test with Supabase client


## Environment Variables Template

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key_here
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key_here

# OpenAI
OPENAI_API_KEY=sk-your_api_key_here

# Google OAuth
NEXT_PUBLIC_GOOGLE_CLIENT_ID=your_google_client_id_here

# App
NEXT_PUBLIC_APP_URL=http://localhost:3000
NODE_ENV=development
```
